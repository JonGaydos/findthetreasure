Three Architecture Approaches

Pick the stack that best fits the app. All three use Leaflet.js, shadcn/ui, and a single Docker container.

A

Next.js Full-Stack

Single framework — React UI + API routes all in one. API routes handle encryption/decryption server-side; Leaflet runs client-side. One Docker container, no separate backend service.

Pros

  • Simplest Docker setup (one container, one port)
  • shadcn/ui is built for Next.js — best integration
  • No context switching between languages
  • API routes = coordinates never touch the browser
  • No database needed — stateless via encrypted tokens

Cons

  • Less familiar if you haven't used Next.js API routes
  • Slightly heavier than a bare Express server

Recommended — cleanest fit for this use case

B

FastAPI (Python) + React

Two services: a Python FastAPI backend handles all crypto and distance math, a React frontend handles the UI. Both built into one Docker image via a multi-stage build.

Pros

  • Matches your Drone Unit Manager stack (familiar)
  • Python's cryptography library is excellent
  • Clear separation of concerns

Cons

  • Two processes in Docker (FastAPI + nginx or serve)
  • More boilerplate for a relatively simple app
  • CORS config between frontend and backend
C

Node.js/Express + React

Express handles encryption, distance math, and serves the React build. Pure JavaScript end-to-end. One Docker container, Express serves both the API and the static frontend.

Pros

  • JS all the way through — simple mental model
  • One container, one process
  • Lightweight and fast to build

Cons

  • More manual setup vs Next.js
  • shadcn/ui requires more config outside of Next.js
  • No SSR benefits (minor for this app)
